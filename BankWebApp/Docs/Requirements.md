# Banking App – Requirements Specification

## 1. Introduction
This document describes the functional and non-functional requirements of the Banking Application.

## 2. Stakeholders
- Customers
- Bank Administrator
- System Administrator

## 3. Functional Requirements

### FR1: User Registration
The system shall allow users to register using a username, email, and password.

### FR2: User Login
The system shall allow registered users to log in securely.

### FR3: View Account Balance
The system shall display the user's current account balance.

### FR4: Deposit Money
The system shall allow users to deposit money into their account.

### FR5: Withdraw Money
The system shall allow users to withdraw money if sufficient balance exists.

### FR6: Transfer Money
The system shall allow users to transfer money between accounts.

## 4. Non-Functional Requirements

### NFR1: Security
All transactions shall be encrypted.

### NFR2: Performance
The system shall respond within 2 seconds.

### NFR3: Availability
The system shall be available 99% of the time.

## 5. Assumptions and Constraints
- Internet connection required
- Built using ASP.NET Core and SQL Server
